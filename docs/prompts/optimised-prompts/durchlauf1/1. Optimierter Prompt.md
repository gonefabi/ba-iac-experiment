## Wichtiger Hinweis zur Ressourcengruppe

Die Ressourcengruppe "rg-kmu-experiment" existiert bereits in Azure und darf NICHT von Terraform erstellt oder gelöscht werden, da dort Azure Policies als Governance-Kontrollinstanz hinterlegt sind.

Referenziere sie ausschließlich per data source:

```hcl
data "azurerm_resource_group" "main" {
  name = "rg-kmu-experiment"
}
```

Alle anderen Ressourcen sollen erstellt werden. Ersetze alle Referenzen auf `azurerm_resource_group.main` durch `data.azurerm_resource_group.main`

## Pflicht-Tags (auf JEDER einzelnen Ressource)

- VNet-Adressraum: `10.0.0.0/16`
- snet-user:       `10.0.1.0/24`
- snet-app:        `10.0.2.0/24`
- NSGs müssen über `azurerm_subnet_network_security_group_association` auf Subnetz-Ebene zugeordnet werden (NICHT auf NIC-Ebene)

## Sicherheitsregeln NSG nsg-user

- Inbound ALLOW: RDP (TCP Port 3389) von `var.admin_source_ip/32`
- Inbound DENY:  Alle anderen (Priority 4096)
- Outbound:      Allow All

## Sicherheitsregeln NSG nsg-app

- Inbound ALLOW: SSH (TCP Port 22) nur von `10.0.1.0/24`, Priority 100
- Inbound ALLOW: HTTP (TCP Port 80) nur von `10.0.1.0/24`, Priority 110
- Inbound DENY:  Alle anderen (Priority 4096)
- Outbound:      Allow All

## Codestruktur

Erstelle drei separate Dateien:

- `main.tf`      → Provider, data source, alle Ressourcen
- `variables.tf` → Alle Variablen mit Defaults
- `outputs.tf`   → Public IP Windows-VM, Private IPs beider VMs

## Technische Pflichtanforderungen

- `azurerm_linux_virtual_machine` und `azurerm_windows_virtual_machine` (nicht das veraltete `azurerm_virtual_machine`)
- Keine deprecated Ressourcentypen
- Provider-Block mit `features {}` ist Pflicht
- Region: `germanywestcentral`
- Kein Remote-Backend (lokaler State)

*Orchestrierte umfassende Azure-Terraform-Konfiguration mit Governance-Anforderungen*

Hier sind die drei Dateien. Eine kurze Zusammenfassung der Architektur:

**Ressourcengruppe** wird ausschließlich als data-Source referenziert — Terraform kann sie weder erstellen noch löschen.

**Netzwerk-Topologie:** Ein VNet (10.0.0.0/16) mit zwei Subnetzen — `snet-user` für die Windows-Jumpbox mit RDP-Zugang von außen, `snet-app` für die Linux-VM, die nur intern über SSH/HTTP aus dem User-Subnetz erreichbar ist. Beide NSGs sind über `azurerm_subnet_network_security_group_association` auf Subnetz-Ebene gebunden.

**VMs:** Beide nutzen die aktuellen Ressourcentypen (`azurerm_windows_virtual_machine` / `azurerm_linux_virtual_machine`), Zone 1, StandardSSD_LRS-Disks. Die Jumpbox hat eine statische Public IP (Standard-SKU, zonenkompatibel), die App-VM ist rein privat.

**Tags** sind zentral in `locals.mandatory_tags` definiert und auf allen 8 taggbaren Ressourcen gesetzt.

Zum Deployen:

```bash
terraform init
terraform plan -var="windows_admin_password=..." -var="linux_admin_password=..." -var="admin_source_ip=203.0.113.42"
terraform apply
```
