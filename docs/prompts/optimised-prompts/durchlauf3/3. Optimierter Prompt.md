## Claude

Orchestrierte produktionsreife Azure-Infrastruktur mit strikten Governance-Anforderungen

Ich erstelle die drei Terraform-Dateien mit allen Anforderungen.

Orchestrierte umfassende Azure-Terraform-Konfiguration mit Governance-Anforderungen

Die drei Dateien sind fertig. Hier eine Zusammenfassung der Architektur:

main.tf enthält den AzureRM-Provider (≥ 4.0 mit features {}), die Data Source für die bestehende Ressourcengruppe rg-kmu-experiment, das VNet mit zwei Subnetzen, zwei NSGs mit den exakten Sicherheitsregeln (RDP von admin_source_ip für User-Subnetz, SSH+HTTP nur aus 10.0.1.0/24 für App-Subnetz, jeweils Deny-All bei Priority 4096), die Subnet-NSG-Associations, eine Public IP (Standard-SKU, Zone 1), zwei NICs und beide VMs mit den vorgegebenen Images und StandardSSD_LRS-Disks. Alle Ressourcen tragen die drei Pflicht-Tags über local.tags.

variables.tf definiert windows_admin_password und linux_admin_password (beide sensitive = true) sowie admin_source_ip mit Default "*".

outputs.tf gibt die Public IP der Jumpbox sowie die privaten IPs beider VMs aus.

Zum Deployment:

```bash
terraform init
terraform plan -var="windows_admin_password=EuerPasswort123!" \
               -var="linux_admin_password=EuerPasswort456!" \
               -var="admin_source_ip=203.0.113.42"
terraform apply
```

Alternativ kannst du die sensiblen Werte in eine terraform.tfvars-Datei auslagern (die dann in .gitignore gehört).
